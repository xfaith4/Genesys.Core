Release 1.4 operator workflow

PowerShell entry points for package export, support handoff, and the full demo path.

Standard package export

Builds a generic operator package with Excel, HTML, Markdown, Elastic bulk payload, and Power BI import bundle outputs.

pwsh -NoProfile -File .\scripts\Export-InvestigationPackage.ps1 -RunFolder .\samples\demo-agent-investigation -DestinationRoot .\samples\release-1.4-packages -Force

Diagnostics handoff

Writes a redacted support bundle and copies it to the clipboard when available.

pwsh -NoProfile -File .\scripts\Copy-InvestigationDiagnosticsBundle.ps1 -RunFolder .\samples\demo-agent-investigation,.\samples\demo-campaign-investigation,.\samples\demo-queue-investigation -OutputPath .\out\investigation-diagnostics.json

Golden-path demo

Regenerates the sample runs, packages, diagnostics, and scenario notes under one folder.

pwsh -NoProfile -File .\scripts\Invoke-GoldenPathDemo.ps1 -Destination .\samples\golden-path-demo -Force

How to load runs

Investigations emit a folder per run under out/<investigationKey>/<runId>/ containing manifest.json, summary.json, events.jsonl, and data/*.jsonl. Drop any combination of those files (or the whole folder) onto this app — the console identifies the run by investigationKey + runId from the manifest.
